Order

Cria um pedido de forma assíncrona.

Caso o corpo da requisição seja válido, a API retorna um status 202 ACCEPTED e a criação do pedido será feita de forma assíncrona. Caso contrário, a API retorna o status 400 BAD REQUEST.

Authorizations:
bearer-auth
Request Body schema: application/json
required
date
required
string <date> yyyy-MM-dd

Data do pedido, no formato yyyy-MM-dd.

externalReference
required
string

Código único do pedido na base da loja.

required
Array of objects (PublicApiProductCreationDto)

Lista dos produtos que foram comprados nesse pedido. A lista deve conter no mínimo um produto.

vendorCnpj
required
string = 14 characters

CNPJ da loja, no formato de apenas números.

discount
number

Desconto aplicado sobre o valor total do pedido, não relacionado a um produto específico. Utilizando '.' como separador entre reais e centavos.

increase
number

Acréscimo aplicado sobre o valor total do pedido, não relacionado a um produto específico. Utilizando '.' como separador entre reais e centavos.

Responses

Production URL

https://api.auvi.com.br/v1/orders

      }
    ],
  • "vendorCnpj": "75396765000191"
}

Catalog

Cria ou atualiza um produto do catálogo de uma loja, de forma assíncrona.

Caso o corpo da requisição seja válido, a API retorna um status 202 ACCEPTED e a criação ou atualização do produto será feita de forma assíncrona. Caso contrário, a API retorna o status 400 BAD REQUEST.

Authorizations:
bearer-auth
Request Body schema: application/json
required
brand
required
string [ 3 .. 100 ] characters

Nome da marca.

externalReference
required
string

Código único do produto na base da loja.

lastUpdateDate
required
string <date-time> yyyy-MM-dd HH:mm:ss

Data da última atualização desse produto no banco de dados da loja. Será utilizada para validar se o produto deve ser atualizado ou não na base da Auvi.

name
required
string [ 5 .. 255 ] characters

Nome do produto.

sellingUnit
required
string
Enum: "CASE" "UNIT" "WEIGHT"

Unidade de comercialização do produto. Pode ser um entre os valores CASE (produto vendido por caixa), UNIT (produto vendido por unidade) e WEIGHT (produto vendido por peso).

stockQuantity
required
number

Quantidade do produto em estoque. Número inteiro caso o produto seja vendido em unidades, ou valor decimal utilizando '.' como separador entre reais e centavos caso seja vendido em peso.

unitPrice
required
number

Preço unitário do produto. Utilizando '.' como separador entre reais e centavos.

vendorCnpj
required
string = 14 characters

CNPJ da loja, no formato de apenas números.

ean
string = 13 characters

EAN do produto, que é um código universal de 13 dígitos. O campo deve ser passado apenas como dígitos, sem caracteres especiais. Um entre EAN, NCM e GTIN é obrigatório.

gtin
string [ 8 .. 14 ] characters

GTIN do produto, que é um código que pode ter 8, 12, 13 ou 14 dígitos. O campo deve ser passado apenas como dígitos, sem caracteres especiais. Um entre EAN, NCM e GTIN é obrigatório.

ncm
string = 8 characters

NCM do produto, que é um código de 8 dígitos. O campo deve ser passado apenas como dígitos, sem caracteres especiais. Um entre EAN, NCM e GTIN é obrigatório.

Responses

Production URL

https://api.auvi.com.br/v1/products-catalog

Request samples

  • Payload
Content type
application/json
{
  • "brand": "Dux",
  • "ean": "7890000000000",
  • "externalReference": "17202404094215254",
  • "gtin": "7890000000000",
  • "lastUpdateDate": "2019-08-24T14:15:22Z",
  • "name": "Almaromi vinagre organico maca 500ml",
  • "ncm": "78900000",
  • "sellingUnit": "UNIT",
  • "stockQuantity": 10,
  • "unitPrice": 10.25,
  • "vendorCnpj": "75396765000191"
}

Shopper

Cria ou atualiza um shopper, de forma assíncrona.

Caso o corpo da requisição seja válido, a API retorna um status 202 ACCEPTED e a criação ou atualização do shopper será feita de forma assíncrona. Caso contrário, a API retorna o status 400 BAD REQUEST.

Authorizations:
bearer-auth
Request Body schema: application/json
required
externalReference
required
string

Código único do shopper na base da loja.

lastUpdateDate
required
string <date-time> yyyy-MM-dd HH:mm:ss

Data da última atualização desse shopper no banco de dados da loja. Será utilizada para validar se o shopper deve ser atualizado ou não na base da Auvi.

name
required
string [ 5 .. 255 ] characters

Nome do shopper.

vendorCnpj
required
string = 14 characters

CNPJ da loja, no formato de apenas números.

dateOfBirth
string <date> yyyy-MM-dd

Data de nascimento do shopper

documentValue
string [ 11 .. 14 ] characters

Valor do documento do shopper, deve conter apenas dígitos (11 caso o documento seja um CPF ou 14 caso seja um CNPJ).

email
string

Email do shopper.

phone
string [ 12 .. 13 ] characters

Valor do telefone do shopper, deve conter apenas dígitos, no formato de DDI seguido por DDD seguido do número, exemplo 5511987654321.

Responses

Production URL

https://api.auvi.com.br/v1/shoppers

Request samples

Content type
application/json
{

  • "documentValue": "12345678911",
  • "email": "joao@email.com",
  • "externalReference": "17202404094215254",
  • "lastUpdateDate": "2019-08-24T14:15:22Z",
  • "name": "João Silva",
  • "phone": "5511987654321",
  • "vendorCnpj": "75396765000191"
}

Vendor

Cria ou atualiza uma loja, de forma assíncrona.

Caso o corpo da requisição seja válido, a API retorna um status 202 ACCEPTED e a criação ou atualização da loja será feita de forma assíncrona. Caso contrário, a API retorna o status 400 BAD REQUEST.

Authorizations:
bearer-auth
Request Body schema: application/json
required
city
required
string

Cidade da loja.

cnpj
required
string = 14 characters

CNPJ da loja, no formato de apenas números.

personName
required
string [ 5 .. 100 ] characters

Nome do responsável pela loja.

phone
required
string [ 12 .. 13 ] characters

Telefone da loja, deve conter apenas dígitos, no formato de DDI seguido por DDD seguido do número, exemplo 5511987654321.

state
required
string = 2 characters

Estado da loja, no formato de sigla (SP, RJ, RS, etc).

storeName
required
string [ 5 .. 100 ] characters

Razão social da loja.
